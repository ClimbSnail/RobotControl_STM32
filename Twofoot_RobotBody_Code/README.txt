此为双足体感机器人主程序（机器人上的程序）

通过串口传输文本形式的数据即可改变控制板上24个舵机口的PWM
总共分为L1~12 R1~12（其中R1R2L1L2不用 输出有问题）
	传输数据用空格隔开，帧尾加上回车键(0x0D 0x0A)帧格式如(文本)：
	L 1500 1500 1500 1500 1500 1500 1500 1500 1500 1500 1500 1500
	R 1500 1500 1500 1500 1500 1500 1500 1500 1500 1500 1500 1500回车

	动作指令用十六进制传输 格式为：数据+0x0D 0x0A
	
存在的问题：控制板给串口蓝牙的电压为3.3V  引起了蓝牙(标称3.6V-6V)运行的不稳定。
			控制板给语音合成模块的电压同样是3.3v，电压过低导致合成的音量小且合成模块工作不稳定。
			
工程中action.c文件中第15的errorValueL、16行的errorValueR是用来调节舵机安装时候的误差的（值分正负），
在为调整误差值前(即errorValue值全为0)，通过串口发送Debug指令来查看当前舵机安装与理想中位状态的差。
误差值的计算：对应位上的值=1500-使舵机达到中位的值。
最后计算得到的值挨个填入errorValue数组中。
